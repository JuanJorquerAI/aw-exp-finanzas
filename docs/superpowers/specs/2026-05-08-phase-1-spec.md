## Estado global: URL como fuente de verdad

Parámetros de URL:

| Param | Valores | Default |
|-------|---------|---------|
| `company` | `AW` \| `EXPRO` | `AW` |
| `month` | `YYYY-MM` | mes actual |

`AppSidebar` lee y escribe estos params. Todos los componentes los consumen via `useSearchParams`.

---

## Flujo de datos por vista

### Dashboard

1. Lee `?company` y `?month` de la URL
2. Construye `dateFrom` (primer día del mes) y `dateTo` (último día del mes)
3. `useQuery` → `GET /transactions?companyId=X&dateFrom=Y&dateTo=Z`
4. Calcula totales en cliente:
   - Para cada transacción, busca `allocation` donde `allocation.companyId === selectedCompany`
   - Suma `allocation.amountCLP` agrupado por `type` y `status`
5. Renderiza `CompanySummaryCard` por cada empresa + `MonthTotalBar` con suma combinada

### CxC (`/cxc`)

1. `useQuery` → `GET /transactions?companyId=X&type=INCOME&status=PENDING`
2. Renderiza `TransactionTable` con columnas: fecha, descripción, contrapartida, monto, vencimiento
3. Columna final: `MarkPaidButton` por fila

### CxP (`/cxp`)

1. `useQuery` → `GET /transactions?companyId=X&type=EXPENSE&status=PENDING`
2. Mismo `TransactionTable`
3. Columna final: `MarkPaidButton` por fila

### Nueva Transacción (drawer global)

- Botón "+" en sidebar abre `NewTransactionDrawer`
- Form: empresa, tipo, monto, moneda, fecha, descripción, contrapartida (opcional), categoría (opcional)
- Submit: `useMutation` → `POST /transactions` → cierra drawer + `invalidateQueries`
- Phase 1: allocations al 100% para empresa seleccionada (sin UI de split)

## Cambios API requeridos

### 1. Fix: filtro `companyId` en `findAll`

**Archivo:** `apps/api/src/transactions/transactions.service.ts`

```ts
// Antes (incorrecto — filtra por empresa principal, no por allocations)
companyId && { companyId }

// Después (correcto — per arquitectura CLAUDE.md)
companyId && { allocations: { some: { companyId } } }
```

**Por qué:** CLAUDE.md dice "reportes por empresa siempre agregan sobre `transaction_allocations`". Con el filtro actual, transacciones donde la empresa participa como allocation secundaria no aparecen.

### 2. `POST /transactions` — soporte `allocations[]`

**Archivo:** `apps/api/src/transactions/dto/create-transaction.dto.ts`

```ts
class AllocationDto {
  @IsString() companyId: string;
  @IsNumber() @Min(0) @Max(100) percentage: number;
}

class CreateTransactionDto {
  // ... campos existentes ...
  @IsOptional()
  @ValidateNested({ each: true })
  @Type(() => AllocationDto)
  allocations?: AllocationDto[];
}
```

**Archivo:** `apps/api/src/transactions/transactions.service.ts`

```ts
async create(dto: CreateTransactionDto) {
  const { allocations, ...txData } = dto;
  const allocs = allocations ?? [{ companyId: txData.companyId, percentage: 100 }];

  return this.prisma.$transaction(async (tx) => {
    const transaction = await tx.transaction.create({ data: txData });
    await tx.transactionAllocation.createMany({
      data: allocs.map((a) => ({
        transactionId: transaction.id,
        companyId: a.companyId,
        percentage: a.percentage,
        amountCLP: (txData.amountCLP * a.percentage) / 100,
      })),
    });
    return transaction;
  });
}
```

No requiere cambios.
